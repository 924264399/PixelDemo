# Pixel Town Demo - 第四阶段目标规划

## 🎯 核心目标：智能NPC决策系统

让NPC基于时间、环境和玩家行为进行自主决策，从硬编码行为转变为AI驱动的智能体。

## 📋 主要功能目标

### 1. LLM集成 - AI决策引擎
- **集成OpenAI GPT API**或本地LLM
- NPC根据当前状态（时间、位置、玩家距离）做出决策
- 智能对话生成，告别"阿巴阿巴"

### 2. 智能行为决策系统
- **自主行为选择**：
  - 工作时间 → 主动去工作地点
  - 休息时间 → 选择娱乐活动
  - 看到玩家 → 决定是否主动交流
- **动态状态切换**：
  - 基于时间的生物钟（早起工作，晚上休息）
  - 基于心情的随机性（偶尔偷懒、主动社交）
  - 基于环境的适应性（下雨进室内，天亮出门）

### 3. 智能对话系统
- **上下文感知对话**：
  - 记住之前的对话内容
  - 根据当前时间和位置调整话题
  - 主动发起不同类型的对话
- **多样化交流**：
  - 工作中：谈论工作、推荐商品
  - 休息时：聊天气、兴趣爱好
  - 夜晚：分享感受、计划明天

## 🔧 技术实现方案

### 1. LangChain Agent架构设计

```typescript
// LangChain基础组件
interface NPCAgent {
  memory: ConversationBufferMemory;      // 对话记忆
  personality: NPCPersonality;           // 人设系统  
  perception: GameStatePerception;       // 游戏感知
  decisionMaker: LLMDecisionEngine;      // LLM决策引擎
  actionExecutor: GameActionExecutor;    // 行为执行器
}

interface NPCPersonality {
  name: string;                 // "小美"
  role: string;                 // "咖啡馆店员" 
  traits: string[];             // ["温和", "勤奋", "略微内向"]
  workHours: { start: number; end: number };
  interests: string[];          // ["咖啡文化", "音乐", "读书"]
  speechStyle: string;          // "温和礼貌，偶尔害羞"
  lazyProbability: number;      // 0.1 (摸鱼概率10%)
}

interface GameStatePerception {
  getCurrentTime(): { hour: number; minute: number };
  getCurrentLocation(): SceneType;
  getPlayerDistance(): number;
  getWeather(): string;         // 为未来扩展预留
  getEnergyLevel(): number;     // 基于时间和活动计算
}

interface NPCDecision {
  action: 'work' | 'rest' | 'socialize' | 'move' | 'continue';
  targetLocation?: { x: number; y: number };
  conversation?: string;
  reasoning: string;
  confidence: number;           // 决策信心度
}
```

### 2. LangChain集成架构

```typescript
// 核心LangChain组件使用
class NPCLangChainAgent {
  private llm: ChatOpenAI;
  private memory: ConversationBufferMemory;
  private personalityPrompt: PromptTemplate;
  private decisionPrompt: PromptTemplate;
  
  constructor(personality: NPCPersonality, apiKey: string) {
    // 初始化LLM
    this.llm = new ChatOpenAI({
      openAIApiKey: apiKey,
      modelName: "gpt-4",
      temperature: 0.7
    });
    
    // 初始化记忆系统
    this.memory = new ConversationBufferMemory({
      memoryKey: "chat_history",
      returnMessages: true
    });
    
    // 人设提示词模板
    this.personalityPrompt = PromptTemplate.fromTemplate(`
      你是{name}，{role}。
      性格特点：{traits}
      工作时间：{workHours}
      兴趣爱好：{interests}
      说话风格：{speechStyle}
      
      请保持角色一致性，用这个人设来回应所有对话。
    `);
  }
}
```

### 3. 分阶段实现计划

Priority 1: 基础架构 + 记忆系统 
核心价值: 建立AI对话的基础，告别"阿巴阿巴"
技术基础: LangChain + ConversationBufferMemory
验证目标: NPC能记住玩家说过的话

🥈 Priority 2: 人设系统 + 感知能力 
核心价值: 建立"小美"这个角色的完整人格
技术基础: NPCPersonality接口 + GameStatePerception
验证目标: 对话风格一致，能感知时间和位置

🥉 Priority 3: 自主决策系统 
核心价值: NPC根据人设和时间自主决定行为
技术基础: LLM决策引擎 + 提示词模板
验证目标: 能自己决定上班/摸鱼，符合人设

🏅 Priority 4: 高级交互 + 主动社交
核心价值: NPC主动找玩家聊天，增强互动
技术基础: 距离感知 + 情绪状态系统
验证目标: 能根据情况主动发起对话

🔮 Priority 5: Agent间通信预备 
核心价值: 为未来多NPC系统做准备
技术基础: 共享知识库 + 消息协议
验证目标: 单Agent能记录和检索信息

🎪 为什么这样排序？
记忆优先: 没有记忆的对话毫无意义
人设第二: 有了记忆，需要一致的人格
决策第三: 有了人格，才能做符合角色的决策
社交第四: 有了决策，才能主动交互
通信最后: 单Agent完善后，再考虑多Agent


优先级	核心代码 / 操作	验证目标（一句话）
1. 记忆系统	初始化ConversationBufferMemory，聊天时saveContext	玩家问 2 个相关问题，NPC 能关联回答

2. 人设 + 感知	写固定 Prompt 模板（含人设 + 时间 / 位置），注入到 LLM 请求	不同时间 / 位置问 “你在干嘛”，回复符合场景

3. 自主决策	LLM 返回 “上班 / 回家”，调用setTargetPosition移动	调时间到 8 点，NPC 自动走向咖啡馆

4. 主动社交	玩家靠近 NPC 时，自动触发预设问题（“要不要喝咖啡？”）	玩家走到 NPC 身边，对话框自动弹出

5. 通信预备	用MemoryVectorStore存 NPC 行为日志，写检索方法	能查到 “小美 8 点去了咖啡馆” 的记录




LLM做自主决策时，加一个「节流规则」不让 NPC 每秒 / 每帧思考，只在关键节点调用 LLM：

整点（8 点、18 点）
切换场景时
玩家靠近时
这样一天 24 分钟的游戏里，NPC 可能只调用 3-5 次 LLM，消耗极低。
