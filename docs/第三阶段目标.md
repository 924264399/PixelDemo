# Pixel Town Demo - 第三阶段目标

## 阶段概述
在第二阶段成功实现相机跟随、屋顶透明度渐变和碰撞检测的基础上，第三阶段将重点添加 NPC 角色的完整交互功能，包括序列帧动画、对话系统和基础寻路能力，为后续的AI Agent集成奠定基础。

## 核心功能目标

### 1. NPC序列帧动画系统
- **角色动画**：为NPC实现行走、站立等基础序列帧动画
- **方向适配**：根据移动方向自动切换对应的动画序列
- **平滑过渡**：动画之间的平滑切换，避免突兀变化
- **资源管理**：高效加载和管理动画帧资源

### 2. NPC对话系统（基础版）
- **对话气泡**：点击NPC时显示对话气泡UI
- **文字输入**：玩家可以输入文字与NPC进行对话
- **消息显示**：支持双向消息显示（玩家消息 + NPC回复）
- **交互控制**：
  - 点击NPC触发对话
  - 按ESC或点击外部关闭对话
  - 回车发送消息
- **预设回复**：NPC基于简单规则生成预设回复（不接入LLM）

### 3. NPC基础寻路系统
- **路径点系统**：预设关键位置点（咖啡馆、便利店、公园、家）
- **自动巡逻**：NPC能在指定地点间自动移动
- **碰撞避让**：移动过程中避开障碍物
- **平滑移动**：使用缓动函数实现自然的移动效果
- **示例路线**：咖啡馆 → 便利店 → 公园 → 家 → 咖啡馆（循环）

## 技术实现规划

### 目录结构调整
```
src/
├── game/
│   ├── scenes/
│   │   └── MainScene.ts (扩展支持NPC)
│   └── characters/
│       ├── Player.ts
│       └── NPC.ts (新增完整NPC类)
├── agents/
│   └── npc-agents/ (预留AI Agent位置)
├── store/
│   └── dialogStore.ts (新增对话状态管理)
└── ui/
    └── DialogBox/
        └── DialogBubble.tsx (新增对话气泡组件)
```

### 关键技术点
- **Phaser.js 动画系统**：使用 `this.anims.create()` 创建序列帧动画
- **Zustand 状态管理**：管理对话状态、输入内容、消息历史
- **React UI 组件**：实现响应式的对话气泡界面
- **简单路径规划**：使用预设路径点 + 线性插值，避免复杂算法
- **TypeScript 类型安全**：定义清晰的接口和类型

- **动画系统**：支持更复杂的AI行为表现
- **对话框架**：可直接集成LLM API

## 风险评估与应对
- **性能风险**：多个动画同时运行可能影响性能 → 优化动画资源，限制同时活跃NPC数量
- **UI复杂度**：对话系统可能变得复杂 → 保持第三阶段功能简单，专注于基础框架
- **寻路精度**：简单路径可能不够智能 → 先实现基础功能，后续再优化算法
