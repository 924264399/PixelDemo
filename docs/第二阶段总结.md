## 技术方案详情

### 1. 遮罩图系统架构

#### 遮罩图规格
- **尺寸**: 512x512像素（对应2048x2048世界，比例1:4）
- **collision_mask.png**: 白色=碰撞体，黑色=可通行
- **indoor_outdoor_mask.png**: 黑色=室内，白色=室外

#### 坐标转换公式
```typescript
// 世界坐标 → 遮罩坐标
const maskX = Math.floor(worldX / 4);
const maskY = Math.floor(worldY / 4);
```

### 2. 相机跟随实现
```typescript
// 启用相机跟随
this.cameras.main.startFollow(this.player, true, 0.1, 0.1);
this.cameras.main.setBounds(0, 0, 2048, 2048);
```

### 3. 屋顶透明度控制逻辑
- **检测频率**: 每帧检测玩家位置
- **透明度目标**:
  - 室内: targetAlpha = 0.0
  - 室外: targetAlpha = 1.0
- **平滑过渡**: `currentAlpha += (targetAlpha - currentAlpha) * 0.02`

### 4. 碰撞检测算法
- **预测式检测**: 先计算目标位置，再检测是否碰撞
- **阈值设置**:
  - 碰撞检测: RGB总和 > 500（白色）
  - 室内检测: RGB总和 < 100（黑色）
- **响应机制**: 如果目标位置有碰撞，保持当前位置不变

## 主要坑点与解决方案

### 🚫 坑点1: 玩家无法移动
**问题描述**:
- 在update方法中先设置速度，后又调用`setVelocity(0)`重置
- 导致键盘输入被覆盖，玩家完全无法移动

**根本原因**:
- 碰撞检测逻辑放在了错误的位置
- 没有区分"当前速度"和"目标位置"

**解决方案**:
- 采用预测式碰撞检测
- 先计算期望的目标位置
- 再检测目标位置是否安全
- 只有安全才应用移动

### 🚫 坑点2: 遮罩图初始化失败
**问题描述**:
- 直接使用Phaser纹理无法获取ImageData
- 需要创建离屏Canvas来处理像素数据

**解决方案**:
- 创建独立的HTML Canvas元素
- 使用`drawImage()`将纹理绘制到Canvas
- 通过`getImageData()`获取像素数据

### 🚫 坑点3: 坐标边界检查
**问题描述**:
- 玩家靠近世界边缘时，遮罩坐标可能越界
- 导致数组访问错误或undefined

**解决方案**:
- 添加完整的边界检查
- `if (maskX >= 0 && maskX < maskWidth && maskY >= 0 && maskY < maskHeight)`

### 🚫 坑点4: 性能优化
**问题描述**:
- 每帧都进行像素级检测可能影响性能
- 特别是在低端设备上

**优化措施**:
- 遮罩图使用较小尺寸（512x512）
- 坐标转换使用整数运算
- 避免重复创建对象
