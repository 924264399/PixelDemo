# 屋顶透明度渐变功能问题分析

## 当前状态
- ✅ **直接变透明功能**：工作正常
- ❌ **渐变透明功能**：存在实现问题

## 问题描述
在尝试实现屋顶透明度渐变效果时，发现以下问题：

### 1. **逻辑冲突问题**
- 平滑过渡逻辑与直接设置逻辑存在冲突
- 在同一帧中，先重置所有屋顶为不透明，再应用平滑过渡，导致效果被覆盖

### 2. **状态管理复杂性**
- 需要同时维护 `currentAlpha` 和 `targetAlpha` 两个状态
- 状态同步容易出错，特别是在多屋顶场景下

### 3. **性能考虑**
- 每帧都需要计算所有屋顶的透明度插值
- 虽然性能开销不大，但增加了代码复杂度

## 已尝试的解决方案

### 方案A：纯平滑过渡
```typescript
// 问题：逻辑冲突，效果被覆盖
private updateRoofTransparencyByPixelDetection() {
    // 重置目标透明度
    roofKeys.forEach(key => this.targetRoofAlphas.set(key, 1.0));

    // 检测并设置目标透明度
    roofKeys.forEach(key => {
        if (this.hasRoofPixelAt(playerX, playerY, key)) {
            this.targetRoofAlphas.set(key, 0.0);
        }
    });

    // 平滑过渡
    roofKeys.forEach(key => {
        // 插值计算...
        roofSprite.setAlpha(newAlpha);
    });
}
```

### 方案B：混合模式（当前使用）
- 直接设置透明度，跳过平滑过渡
- ✅ 简单可靠
- ✅ 性能最佳
- ❌ 无渐变效果

### 预期实现
```typescript
private updateRoofTransparencySmooth() {
    const playerX = Math.floor(this.player.x);
    const playerY = Math.floor(this.player.y);
    const roofKeys = ['roof_home', 'roof_cafe', 'roof_store'];

    // 只更新目标透明度，不直接设置
    roofKeys.forEach(key => {
        const hasRoof = this.hasRoofPixelAt(playerX, playerY, key);
        this.targetRoofAlphas.set(key, hasRoof ? 0.0 : 1.0);
    });

    // 统一进行平滑过渡
    roofKeys.forEach(key => {
        const current = this.currentRoofAlphas.get(key) || 1.0;
        const target = this.targetRoofAlphas.get(key) || 1.0;

        if (Math.abs(current - target) > 0.01) {
            const newAlpha = current + (target - current) * this.roofAlphaSpeed;
            this.currentRoofAlphas.set(key, newAlpha);
            this.roofSprites.get(key)?.setAlpha(newAlpha);
        } else {
            // 精确收敛
            this.currentRoofAlphas.set(key, target);
            this.roofSprites.get(key)?.setAlpha(target);
        }
    });
}
```

### 参数调优
- `roofAlphaSpeed`: 控制渐变速度
  - `0.1`: 快速渐变（约10帧完成）
  - `0.05`: 中等渐变（约20帧完成）
  - `0.02`: 缓慢渐变（约50帧完成）
