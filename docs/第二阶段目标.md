# Pixel Town Demo - 第二阶段目标

## 阶段概述
在第一阶段成功实现基础游戏框架和玩家角色移动的基础上，第二阶段将重点添加 NPC 角色和基础 AI 行为，为后续的完整对话系统和社交传播机制奠定基础。

## 核心功能目标

### 1. NPC 角色创建与管理
- **场景布局**：在十字路口布局中添加 4 个主要地点的 NPC
  - 主角家（Home）
  - 便利店（Convenience Store）
  - 公园（Park）
  - 咖啡馆（Café）
- **角色外观**：为每个 NPC 设计独特的像素风格外观
- **位置管理**：NPC 在各自指定区域内活动，不会离开所属区域

### 2. NPC 基础行为系统
- **自主移动**：NPC 能够在指定区域内随机或按预设路径移动
- **碰撞检测**：NPC 之间以及与环境物体存在物理碰撞
- **行为循环**：实现简单的日常行为模式（如在咖啡馆内走动、在公园长椅休息等）

### 3. 基础 AI Agent 框架
- **Agent 结构**：使用 LangChain.js 构建 NPC Agent 基础类
- **记忆模块**：实现基础记忆系统，存储以下信息：
  - NPC 基本信息（姓名、职业、性格）
  - 预设性格特征（友善、冷漠、热情、幽默等）
  - 简单的背景故事
- **性格系统**：每个 NPC 具有独特的性格特征，影响其行为和对话风格

### 4. 交互触发机制
- **接近检测**：玩家靠近 NPC 时显示交互提示（如 "E" 键提示）
- **点击交互**：支持鼠标点击 NPC 触发对话
- **基础对话框**：实现简单的 UI 对话框组件
- **对话内容**：基于 NPC 性格生成预设的问候语和基础对话

## 技术实现规划

### 目录结构调整
```
src/
├── game/
│   ├── scenes/
│   │   └── MainScene.ts (扩展支持多角色)
│   └── characters/
│       ├── Player.ts
│       └── NPC.ts (新增)
├── agents/
│   ├── npc-agents/
│   │   ├── HomeNPC.ts
│   │   ├── StoreNPC.ts
│   │   ├── ParkNPC.ts
│   │   └── CafeNPC.ts
│   └── base/
│       └── BaseAgent.ts (新增基础Agent类)
├── store/
│   └── npcStore.ts (新增NPC状态管理)
└── ui/
    └── DialogBox/
        └── SimpleDialog.tsx (新增基础对话框)
```

### 关键技术点
- **Phaser.js 扩展**：修改 MainScene 以支持多个动态角色
- **LangChain.js 集成**：创建可扩展的 Agent 基类
- **Zustand 状态管理**：管理 NPC 位置、状态和交互状态
- **TypeScript 类型安全**：定义清晰的接口和类型

## 风险评估与应对
- **性能风险**：多个 NPC 同时运行可能影响性能 → 优化物理计算，限制同时活跃 NPC 数量
- **复杂度风险**：AI Agent 系统过于复杂 → 保持第二阶段功能简单，专注于基础框架
- **集成风险**：Phaser 与 LangChain 集成问题 → 先实现模拟数据，再逐步集成真实 AI 功能
