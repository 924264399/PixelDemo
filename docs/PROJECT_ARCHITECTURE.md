# Pixel Town Demo - AI小镇游戏项目

## 项目概述

这是一个简化的2D像素风AI小镇游戏demo，玩家可以控制主角在小镇中移动并与NPC互动。NPC都是具备完整AI能力的Agent，能够进行记忆、反思和计划，并在玩家与之对话后将信息传播给其他NPC。每个NPC都有独特的性格特征，会对玩家的行为做出不同的反应（如安慰、嘲讽等）。

### 游戏场景
- **十字路口布局**：包含主角家、便利店、公园、咖啡馆四个主要地点
- **角色移动**：所有角色（包括玩家和NPC）都可以在场景中自由移动
- **碰撞检测**：角色之间和环境物体存在物理碰撞
- **对话系统**：点击NPC触发对话框，支持多轮对话

### 核心功能
- 玩家与NPC的自然语言对话
- NPC间的社交传播机制
- 基于性格的差异化反应系统
- 实时2D像素风游戏体验

## 技术栈架构

### 核心技术

#### 1. 游戏引擎 - Phaser.js
- **职责**: 处理2D游戏的核心功能
- **功能**:
  - 像素完美渲染
  - 物理引擎和碰撞检测
  - 精灵动画和场景管理
  - 输入处理（键盘/鼠标控制）
- **优势**: 专为2D游戏设计，Web原生支持，成熟的社区生态

#### 2. AI Agent架构 - LangChain.js
- **职责**: 实现NPC的智能行为
- **模块**:
  - **记忆模块**: 存储对话历史和重要事件
  - **反思模块**: 分析和总结经验
  - **计划模块**: 制定后续行动策略
  - **工具调用**: 与其他系统交互
  - **Skills系统**: 预留技能系统，支持NPC具备特定能力（如烹饪、园艺、社交等）
- **优势**: 模块化设计，易于扩展，内置LLM集成

#### 3. 状态管理 - Zustand
- **职责**: 统一管理游戏全局状态
- **管理状态**:
  - 角色位置和状态
  - 对话系统状态
  - 游戏进度和配置
  - NPC关系网络
- **优势**: 轻量级，响应式更新，TypeScript友好

### 辅助技术

#### 4. 构建工具 - Vite
- 快速开发服务器
- 即时热重载
- 生产环境优化

#### 5. 样式方案 - Tailwind CSS (可选)
- 快速构建UI组件
- 响应式设计支持

#### 6. 后端服务
- **选项A**: Firebase (推荐用于快速原型)
  - 实时数据库
  - 云函数处理LLM API调用
- **选项B**: 自建Node.js服务
  - 完全控制API逻辑
  - 本地开发更灵活

## 开发流程

1. **环境搭建**: 初始化Vite + TypeScript项目，安装核心依赖
2. **基础游戏框架**: 使用Phaser.js创建基本场景和角色移动
3. **状态管理系统**: 实现Zustand store管理游戏状态
4. **AI Agent基础**: 构建LangChain.js Agent框架
5. **对话系统**: 实现UI对话框和游戏内对话逻辑
6. **NPC社交机制**: 实现信息传播和性格反应系统
7. **整合测试**: 将所有系统整合并进行端到端测试

## 依赖版本建议

- **Phaser.js**: ^3.87.0 (最新稳定版)
- **LangChain.js**: ^0.1.x (根据实际需求选择)
- **Zustand**: ^4.5.x
- **Vite**: ^5.x
- **TypeScript**: ^5.x

## 开发方式

本项目采用**渐进式迭代开发**方式：
- **不一次性实现完整功能**，而是分阶段逐步构建
- **每个阶段都有可运行的demo**，确保核心功能正常工作
- **优先实现MVP（最小可行产品）**，再逐步添加复杂功能
- **第一阶段目标**：玩家角色在场景中基本移动
- **后续阶段**：逐步添加NPC、AI行为、对话系统等

这种开发方式确保项目始终处于可演示状态，降低开发风险，便于调试和展示。

这个架构设计确保了各层职责清晰，便于开发、测试和维护，同时为后续的功能扩展提供了良好的基础。
